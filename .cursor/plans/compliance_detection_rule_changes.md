# コンプライアンス検知ルール変更履歴

このドキュメントは、コンプライアンス検知ルールの手動変更履歴を記録します。

## 2025-11-26: 誤検知削減のための辞書ルール削除

### 背景
業務上頻繁に使う言葉（「外部」「社外」）が誤検知の原因となっており、
健全な日報やビジネスコミュニケーションで不要なアラートが発生していた。

### 削除したルール

#### 1. 「外部」ルール
- **ルールID**: 6506
- **パターン**: 外部
- **カテゴリ**: others（情報漏洩・コンプライアンス違反）
- **スコア**: 70点
- **削除理由**: 「外部連絡」「外部レビュー」「外部ミーティング」など、業務上の正常な文脈で頻繁に使用される

#### 2. 「社外」ルール
- **ルールID**: 6507
- **パターン**: 社外
- **カテゴリ**: others（情報漏洩・コンプライアンス違反）
- **スコア**: 95点
- **削除理由**: 「社外ミーティング」「社外メンバー」「社外連絡」など、業務上の正常な文脈で頻繁に使用される

### 削除方法
```bash
# 削除実行（2025-11-26）
bundle exec rails runner "Compliances::DetectionRule.find(6506).destroy!"
bundle exec rails runner "Compliances::DetectionRule.find(6507).destroy!"
```

### 効果
- 日報やビジネスコミュニケーションでの誤検知が大幅に削減
- ルールスコアが不要に高くなることを防止
- AI判定の精度がより反映されるようになった

### 参考
これらのキーワードは文脈依存性が高く、単語単体では違反か否か判断できないため、
AIによる文脈判断に委ねる方が適切と判断。
